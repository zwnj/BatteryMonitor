# BatteryMonitor3 新機能計画 (New Features Plan)

## 1. システムトレイアプリケーション化 (System Tray Application Feature)

### 目的
アプリケーションをWindowsシステムトレイに常駐させ、必要な時だけポップアップ表示されるようにすることで、利便性を向上させます。

### 提案するユーザーエクスペリエンス (UX)
ユーザーの要望に基づき、ホバー（カーソルを当てる）で表示する方式を採用します。

*   **起動時:** メインウィンドウは表示せず、システムトレイにバッテリーアイコンを表示します。
*   **ホバーして表示:**
    *   トレイアイコンの上にカーソルを **1秒間** 保持すると、メインウィンドウがポップアップ表示されます。
    *   カーソルがすぐにアイコン上から離れた場合は、表示されません。
*   **非表示（遅延あり）:**
    *   ウィンドウがフォーカスを失うと（ユーザーが他の場所をクリックするなど）、**1秒間のタイマー**が開始されます。
    *   1秒以内にウィンドウが再びフォーカスを取り戻さなかった場合、ウィンドウは自動的に非表示になります。
*   **右クリック:** 「終了」メニューを含むコンテキストメニューを表示します。

### 技術的な実装計画

プロジェクトに既に追加されている `Hardcodet.NotifyIcon.Wpf` ライブラリを利用して、WPFの作法に沿った形で実装を進めます。

1.  **XAMLでのTaskbarIcon定義:**
    *   `MainWindow.xaml` に、`Hardcodet.NotifyIcon.Wpf` ライブラリの `TaskbarIcon` コンポーネントを定義します。
2.  **ウィンドウの初期化と挙動制御:**
    *   `MainWindow` の `ShowInTaskbar` プロパティを `False` に設定します。
    *   `MainWindow` が起動時に非表示になるようにします。
    *   `MainWindow` の `Closing` イベントをハンドルし、ウィンドウを閉じる代わりに `Hide()` するようにします。
    *   `MainWindow` の `Deactivated` イベントをハンドルし、**非表示タイマーを開始**するようにします。
    *   `MainWindow` の `Activated` イベントをハンドルし、**非表示タイマーを停止**するようにします。
3.  **イベントハンドラとタイマーロジック:**
    *   **表示用タイマー:** `TrayMouseMove` で開始、`TrayMouseLeave` で停止し、遅延表示を実現します。
    *   **非表示用タイマー:** `Deactivated` で開始、`Activated` で停止し、遅延非表示を実現します。
    *   `TaskbarIcon` のコンテキストメニューに「終了」メニューアイテムを定義し、その `Click` イベントでアプリケーションを終了させます。
4.  **アイコンファイルの準備:**
    *   `battery.ico` をプロジェクトのリソースとして組み込みます。
