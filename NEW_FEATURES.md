# BatteryMonitor3 新機能計画 (New Features Plan)

## 1. システムトレイアプリケーション化 (System Tray Application Feature)

### 目的
アプリケーションをWindowsシステムトレイに常駐させ、必要な時だけポップアップ表示されるようにすることで、利便性を向上させます。

### 提案するユーザーエクスペリエンス (UX)
ユーザーの要望に基づき、ホバー（カーソルを当てる）で表示する方式を採用します。

*   **起動時:** メインウィンドウは表示せず、システムトレイにバッテリーアイコンを表示します。
*   **ホバーして表示:**
    *   トレイアイコンの上にカーソルを **1秒間** 保持すると、メインウィンドウがポップアップ表示されます。
    *   カーソルがすぐにアイコン上から離れた場合は、表示されません。
*   **非表示:** ウィンドウ以外の場所をクリックするなどして、ウィンドウがフォーカスを失うと、自動的に非表示になります。
*   **右クリック:** 「終了」メニューを含むコンテキストメニューを表示します。

### 技術的な実装計画

この機能を実現するために、以下の手順で進めます。

1.  **プロジェクト設定の変更:**
    *   `.csproj` ファイルを編集し、Windows Formsの `NotifyIcon` を利用できるようにします。具体的には `<UseWindowsForms>true</UseWindowsForms>` を追加します。
2.  **NotifyIconの初期化:**
    *   `MainWindow.xaml.cs` (または専用のヘルパークラス) に `System.Windows.Forms.NotifyIcon` のインスタンスを作成します。
    *   アイコンファイル、`Visible`プロパティを設定します。
3.  **ウィンドウの初期化と挙動制御:**
    *   `MainWindow` の `ShowInTaskbar` プロパティを `False` に設定します。
    *   `MainWindow` が起動時に非表示になるようにします。
    *   `MainWindow` の `Closing` イベントをハンドルし、ウィンドウを閉じる代わりに `Hide()` するようにします。
    *   `MainWindow` の `Deactivated` イベントをハンドルし、フォーカスを失ったら自動的に `Hide()` するようにします。
4.  **イベントハンドラの追加:**
    *   `System.Windows.Forms.Timer` を利用して、ホバーの遅延表示ロジックを実装します。
        *   `NotifyIcon.MouseMove`: タイマーを開始します。
        *   `NotifyIcon.MouseLeave`: タイマーを停止・リセットします。
        *   `Timer.Tick`: メインウィンドウを表示する処理を実行します。
    *   `NotifyIcon` に `ContextMenuStrip` を設定し、「終了」メニューアイテムを追加します。
    *   「終了」メニューアイテムの `Click` イベントで `Application.Current.Shutdown()` を呼び出し、アプリケーションを終了させます。
5.  **アイコンファイルの検討:**
    *   専用の `.ico` ファイルをプロジェクトに追加するか、既存のアプリケーションアイコンを適切な形式で利用する方法を検討します。
